# Quick Reference Guide

## Getting Started (60 seconds)

### Step 1: Setup Environment
```bash
# Windows
setup.bat

# Linux/Mac
bash setup.sh
```

### Step 2: Configure API Keys
```bash
# Edit .env file
GEMINI_API_KEY=AIzaSy_your-key-here
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your-key-here
```

### Step 3: Run the App
```bash
python app.py
```

### Step 4: Open Browser
```
http://localhost:5000
```

## Common Commands

### Virtual Environment
```bash
# Activate (Windows)
venv\Scripts\activate

# Activate (Linux/Mac)
source venv/bin/activate

# Deactivate
deactivate
```

### Install/Update Packages
```bash
pip install -r requirements.txt
pip install --upgrade requests
```

### Run Application
```bash
python app.py              # Run normally
python app.py --debug      # Enable debug mode
```

### Test Health Check
```bash
curl http://localhost:5000/health
```

### Process Sample JD
```bash
curl -X POST http://localhost:5000/process \
  -F "files=@sample_jd.txt"
```

## Database

### Create Table
Run in Supabase SQL editor:
```sql
-- See supabase_setup.sql for full setup
CREATE TABLE job_extractions (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  role VARCHAR(255),
  skills TEXT[],
  seniority VARCHAR(50),
  location VARCHAR(255),
  salary VARCHAR(255),
  raw_jd TEXT,
  source VARCHAR(255),
  processed_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Verify Connection
```bash
curl http://localhost:5000/health
# Check "database": "connected"
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| "AI processor not available" | Check GEMINI_API_KEY in .env |
| "Table not found" | Run supabase_setup.sql in Supabase |
| "Connection refused" | Run `python app.py` |
| "File rejected" | Use PDF or TXT format (< 16MB) |
| "Empty results" | Check API key and Supabase setup |

## Performance Tips

1. **Batch Processing**: Upload multiple files at once
2. **Monitor Costs**: Track OpenAI API usage
3. **Set Limits**: Configure API rate limits in OpenAI account
4. **Database**: Indexes created on role, created_at, seniority
5. **Caching**: Review extraction history in Supabase
